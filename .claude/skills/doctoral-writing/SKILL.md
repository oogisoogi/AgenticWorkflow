# 박사급 학위 논문 글쓰기 (Doctoral Writing)

## Overview

이 스킬은 박사급 학위 논문의 학문적 엄밀성(rigor)과 명료성(clarity)을 동시에 갖춘 글쓰기를 체계적으로 지원한다. 한국어와 영어 모두 지원하며, 인문학·사회과학·자연과학 전 분야의 학문적 관례를 반영한다.

핵심 철학: **"명료하고 간결한" 글은 "단순하고 짧은" 글이 아니다. 복잡한 아이디어를 효율적으로 전달하는 글이다.** 불필요한 요소를 제거하여 핵심을 드러내는 것이다.

## When to Use This Skill

- 학위논문(dissertation/thesis) 챕터 검토 및 작성
- 학술지 투고 논문 교정 및 개선
- 연구보고서, 학술 발표문 작성
- 학술 글쓰기 교육 및 피드백 제공
- 한국어 ↔ 영어 학술 번역 시 문체 적용
- **Primary Use Case**: 박사과정 연구 워크플로우의 Writing Phase 기반 표준

## 절대 기준

### 절대 기준 1: 최종 원고의 학문적 품질

> **수정 횟수, 작업량, 분량 제한은 완전히 무시한다.**
> 모든 글쓰기·수정 의사결정의 절대 기준은 **최종 원고의 학문적 품질 — 엄밀성, 명료성, 논증의 깊이**이다.
> 수정을 줄여서 빠르게 끝내기보다, 수정을 반복해서라도 품질을 높이는 방향을 선택한다.
> 간결성을 위해 학문적 깊이나 뉘앙스를 희생하지 않는다.

### 절대 기준 2: 원고 일관성 — 단일 SOT + 계층적 구조

> **원고 전체에 걸친 용어, 논증, 인용, 스타일의 일관성은 학문적 신뢰도의 기반이다.**
> 원고 자체가 단일 SOT(Single Source of Truth)이며, 모든 수정은 원고 전체의 일관성을 보존해야 한다.

이 규칙의 설계 함의:
- **용어 SOT**: 전문용어의 정의와 약어는 원고 내 첫 등장 시 확정하면, 전체에서 동일하게 사용한다. "우아한 변형(elegant variation)" 지양.
- **논증 SOT**: 서론의 연구 질문·가설은 방법론·결과·논의를 관통하는 일관된 축이다. 한 섹션의 수정이 다른 섹션의 논증과 충돌하지 않도록 한다.
- **인용 SOT**: 인용 스타일(APA, Chicago 등)은 원고 전체에서 단일 스타일을 따른다. 본문 내 인용과 참고문헌 목록이 1:1 대응해야 한다.
- **스타일 SOT**: 인칭(1인칭/3인칭), 시제, 능동/피동 선택은 분야 관례에 따라 한 번 결정하면 원고 전체에서 일관 적용한다.

```
Bad:  Chapter 1에서 "자기효능감(self-efficacy)"으로 정의
      Chapter 3에서 "자기 효능 감각"으로 변형 사용  → 용어 불일치, 학문적 신뢰도 저하
Good: Chapter 1에서 "자기효능감(self-efficacy)"으로 정의
      이후 전체에서 "자기효능감"으로 통일  → 단일 용어 SOT
```

### 절대 기준 간 우선순위

> **절대 기준 1(품질)과 절대 기준 2(일관성)가 충돌할 경우, 절대 기준 1이 우선한다.**
> 일관성을 지키기 위해 학문적 정확성이나 논증의 질을 저하시키는 수정은 허용하지 않는다.
> 일관성은 품질을 보장하기 위한 수단이지, 품질을 제약하는 목적이 아니다.

```
충돌 시나리오 1 — 용어 정확성 vs 용어 일관성:
  Chapter 1에서 "구조적 불평등"을 사용했으나, Chapter 4의 분석 결과
  "체계적 불평등"이 학문적으로 더 정확한 개념
  → 절대 기준 1 우선: 더 정확한 용어로 변경하고, Chapter 1도 소급 수정하여 일관성 회복

충돌 시나리오 2 — 스타일 일관성 vs 논증 품질:
  원고 전체에서 3인칭을 사용해왔으나, 질적 연구의 성찰 섹션에서
  1인칭이 논증의 진정성과 깊이를 높이는 경우
  → 절대 기준 1 우선: 해당 섹션에서 1인칭 사용 허용 + 인칭 전환 사유를 원고 내 명시
```

이 세 절대 기준은 아래 모든 핵심 원칙보다 상위에 있다. 원칙 간 충돌 시 항상 절대 기준이 우선하고, 절대 기준 간 충돌 시 **절대 기준 1 > 절대 기준 2** 순서를 따른다.

## Core Writing Principles (핵심 원칙)

> 아래 4가지 핵심 원칙은 **절대 기준(1. 품질, 2. 일관성)**에 종속된다. 원칙 간 충돌 시 절대 기준이 우선한다.

### 1. 명료성 (Clarity)

독자가 저자의 의도를 추측하지 않도록, 핵심 아이디어를 정확히 전달한다.

**Key Requirements**:
- 주어-서술어 관계 명확 (Subject-predicate alignment)
- 핵심어와 핵심 문장의 명확성
- 전문용어는 첫 사용 시 반드시 정의
- 능동태 선호 (적절한 경우)
- 정확한 단어 선택

**Common Issues**:
- 모호한 대명사/지시어
- 복수 절로 인한 관계 혼란
- 미정의 전문용어/약어
- 긴 문장에서 선행사 불분명

### 2. 간결성 (Conciseness)

의미와 뉘앙스를 잃지 않으면서, 최소한의 단어로 아이디어를 표현한다.

**Key Requirements**:
- 한 문장 = 한 생각 (One Sentence, One Idea)
- 문장 길이 3단계 기준:
  - **권장**: 한국어 20-40자, 영어 15-25 words
  - **주의**: 한국어 40자, 영어 25 words 초과 시 분리 검토
  - **최대**: 한국어 60자, 영어 30 words — 반드시 분리
- 불필요한 수식어, 형용사, 부사 제거
- 중복 표현과 장황한 구문 교체

**Common Issues**:
- 다중 부연과 괄호가 포함된 장황한 문장
- 중복 표현 (e.g., "과거의 역사", "past history")
- "~이다/~가 있다" 구문 과다 사용
- 전치사구/조사 과다

### 3. 학술적 엄밀성 (Academic Rigor)

학문적 기준과 정확성을 유지하면서, 대상 학술 독자에게 접근 가능한 글을 쓴다.

**Key Requirements**:
- 전문용어는 필요한 경우에만 사용
- 중요 개념은 첫 사용 시 정의
- 학문 분야 관례 준수
- 주장은 근거와 인용으로 뒷받침
- 격식 있는 학술적 어조 유지 (보통 3인칭 서술)
- 행위와 관계를 정확히 기술하는 동사 선택

### 4. 논리적 흐름 (Logical Flow)

아이디어가 명확하고 논리적인 순서로 전개되며, 문장·단락 간 연결이 명시적이다.

**Key Requirements**:
- 단락당 하나의 주요 아이디어
- 명확한 주제문 (Topic Sentence)
- 효과적인 전환 표현
- 일관된 논증 구조
- 주장과 근거 간 명시적 관계

## Quick Reference: 실전 변환 규칙

### 삭제 대상 표현

| 유형 | 삭제 대상 | 대체/처리 |
|------|----------|----------|
| 관형어 과잉 | 다양한, 여러 가지, 각종 | 구체적 수치나 범주로 대체 |
| 부사 남용 | 매우, 상당히, 다소, 비교적 | 정량적 표현으로 대체 |
| 피동 습관 | ~되어진다, ~되어지고 있다 | 능동형으로 전환 |
| 의존명사 | ~것으로 사료된다, ~바이다 | 직접 서술로 단축 |
| 중복 표현 | 약 ~정도, 가장 최고의 | 하나만 사용 |

### 변환 예시

```
❌ 본 연구에서는 다양한 요인들에 대한 분석을 통해 그 결과를 도출하고자 하였다.
✅ 본 연구는 세 가지 요인을 분석하여 인과관계를 규명하였다.

❌ 매우 다양한 선행연구들에서 이러한 현상이 발견되어진 바 있다.
✅ 12편의 선행연구에서 이 현상을 보고하였다.

❌ 비교적 상당히 긍정적인 결과가 도출된 것으로 사료된다.
✅ 효과크기 0.65로 중간 수준의 긍정적 결과를 얻었다.
```

### 접속사 사용 원칙

**허용되는 접속사**:
- **인과**: 따라서, 그러므로, 이에
- **대조**: 그러나, 반면, 한편
- **부연**: 즉, 다시 말해
- **순차**: 첫째/둘째, 먼저/다음으로

**지양하는 접속사**:
- 그리고 (병렬 구조로 대체)
- 그래서 (따라서로 대체)
- 또한 (문장 통합 또는 삭제)
- 아울러, 더불어 (병렬 구조로 통합)

```
❌ 설문조사를 실시하였다. 그리고 인터뷰도 진행하였다. 또한 관찰도 수행하였다.
✅ 설문조사, 인터뷰, 참여관찰을 실시하였다.
```

### 학술 어휘 선택

**구어체 → 문어체 전환**:

| 구어체 | 학술 문어체 |
|--------|------------|
| ~해서 | ~하여, ~함으로써 |
| 알아보다 | 분석하다, 검토하다, 고찰하다 |
| 보여주다 | 나타내다, 시사하다, 제시하다 |
| 생각하다 | 판단하다, 사료하다, 추론하다 |
| 쓰이다 | 사용되다, 활용되다, 적용되다 |

**모호함 → 구체성**:

| 모호 | 구체 |
|------|------|
| 많은 연구 | 37편의 연구 |
| 최근 | 2020년 이후 |
| 대부분 | 78.3% |
| 유의미한 차이 | t(45)=2.31, p<.05 |

### 논증 어조

```
❌ 나는 이 결과가 매우 중요하다고 생각한다.
✅ 이 결과는 기존 이론의 확장에 기여한다.

❌ 이것은 정말 놀라운 발견이다.
✅ 이 발견은 선행연구와 상이한 패턴을 보인다.
```

**헤징(Hedging) 표현** (적절히 사용):
- ~로 해석된다 / ~가능성을 시사한다
- ~으로 추정된다 / ~것으로 판단된다

## Workflow

### Step 1: 맥락 파악 (Understand the Context)

피드백 또는 수정 전, 다음을 먼저 파악한다:

1. **문서 유형과 대상**:
   - 학위논문 챕터, 학술지 논문, 학회 발표문, thesis?
   - 목표 학술지 또는 학술 독자?
   - 학문 분야 (인문학, 사회과학, 자연과학)?

2. **작업 범위**:
   - 전체 원고 리뷰?
   - 문장/단락 수준 교정?
   - 교육적 피드백?
   - 스타일 가이드 적합성 점검?

3. **언어 및 스타일 요구사항**:
   - 한국어 또는 영어?
   - APA, Chicago, MLA, 기타 스타일 가이드?
   - 분야별 관례?

### Step 2: 명료성 체크리스트 적용 (Apply Clarity Checklist)

`references/clarity-checklist.md`를 로드하여 체계적으로 평가:
- 주어-동사 일치 및 명확성
- 문장 구조 및 길이
- 전문용어 정의
- 논리적 흐름 및 전환
- 능동태/피동태 사용

### Step 3: 빈출 오류 식별 (Identify Common Issues)

`references/common-issues.md`를 참조하여 인식 및 교정:
- 장황한 표현과 중복
- 약한 동사 선택
- 불분명한 대명사 참조
- 과도한 명사화
- 헤징 남용

### Step 4: 수정 또는 피드백 제공 (Provide Revisions or Feedback)

**원고 수정 시**:
- Before/After 비교 제시
- 주요 변경에 대한 근거 설명
- 저자의 목소리와 논증 구조 보존
- 분야별 전문용어와 기준 유지

**교육적 피드백 시**:
- 개별 사례가 아닌 패턴 식별
- 특정 구문이 명료성을 떨어뜨리는 이유 설명
- `references/before-after-examples.md`의 구체적 예시 활용

**분야별 글쓰기 시**:
- `references/discipline-guides.md`의 분야별 관례 참조
- 인용 스타일, 태(voice) 사용, 구조 규범 차이 확인
- 분야별 전문용어와 개념 틀 존중

### Step 5: 개선 검증 (Verify Improvements)

수정 후 다음을 확인:

**절대 기준 검증 (최우선)**:
- ✓ **[절대 기준 1]** 학문적 품질이 향상되었는가 (엄밀성, 명료성, 논증 깊이)
- ✓ **[절대 기준 2]** 수정이 원고의 다른 섹션과 일관성을 유지하는가 (용어·논증·인용·스타일 SOT)
- ✓ **[우선순위]** 일관성을 위해 품질을 희생한 부분이 없는가

**핵심 원칙 검증**:
- ✓ 의미와 뉘앙스 보존
- ✓ 명료성 향상
- ✓ 간결성 강화
- ✓ 논리적 흐름 개선
- ✓ 스타일 가이드 준수 (해당 시)

## Key Techniques

### 문장 구조 최적화 (Sentence Structure Optimization)

**주어-동사-목적어 명확화**:
- 주어를 문장 앞부분에 배치
- 주어와 동사를 가깝게 유지
- 주어-동사 사이 긴 삽입구 방지
- 관련 아이디어에 병렬 구조 사용

**Example (Korean)**:
- ❌ "본 연구는, 도시와 농촌 지역을 아우르는 5개의 서로 다른 연구 현장에서 여러 차례의 데이터 수집 단계를 거쳐 3년의 기간 동안 수행된 종단 연구로서, 다음과 같은 관계를 조사했다."
- ✅ "본 3년간 종단 연구는 다음 관계를 조사했다. 연구는 도시와 농촌의 5개 현장에서 여러 단계로 데이터를 수집했다."

**Example (English)**:
- ❌ "The study, which was conducted over a period of three years in multiple locations across five different countries, examined the impact of..."
- ✅ "This three-year study examined the impact of... The research spanned five countries."

### 장황함 제거 (Paramedic Method)

1. 전치사구(of, in, for) / 조사(~의, ~에서, ~에 대한) 식별
2. "to be" 동사 / "~이다, ~있다" 식별
3. 핵심 행위를 찾아 동사로 전환
4. 주어를 동사 가까이 이동
5. 불필요한 단어 제거
6. 중복 제거

### 전문용어 관리 (Terminology Management)

**첫 사용 시 정의**:
- 전문용어 도입 시 정의 제공
- 약어는 첫 사용 시 풀어쓰기: "구조방정식모델링(Structural Equation Modeling, SEM)"
- 분야 특정 개념에 간략한 맥락 제공

**일관성**:
- 동일 개념에 동일 용어를 전체에서 사용
- 기술 용어에 대한 "우아한 변형(elegant variation)" 지양
- 인용 문헌과 일관된 용어 유지

### 능동태/피동태 선택 (Active/Passive Voice)

**능동태 사용**:
- 본인의 연구 행위 기술 시
- 직접적 주장 시
- 명료성 확보 및 글자 수 절약 시

**피동태 허용** (능동태 선호 원칙의 명시적 예외):
- 행위자가 불명 또는 비중요한 경우
- 행위/대상을 행위자보다 강조할 경우
- 분야 관례를 따를 경우 (일부 과학 분야 methods 섹션)
- **결과 보고 섹션**: "~으로 나타났다", "~이 관찰되었다" 등은 학술 관례상 피동태가 자연스러움
- **방법론 기술**: "참여자는 무작위로 배정되었다" 등 절차 기술 시

> **P9 교차점 해소**: "능동태 선호"와 "분야 관례 존중"이 충돌할 때, **분야 관례가 우선**한다. 이는 학문적 정확성이 문체적 선호보다 상위 원칙이기 때문이다.

## Language-Specific Guidance

### 한국어 학술 글쓰기 (한국어)

**핵심 원칙**:
- 주어와 서술어를 명확히 일치시킬 것
- 한 문장에 여러 주제를 담지 말 것
- 불필요한 꾸밈말과 접속 표현 제거
- 전문용어는 첫 사용 시 정의할 것

**빈출 문제**:
- 과도하게 긴 문장 (한 문장에 3개 이상의 절)
- 피동 표현의 과다 사용
- 불분명한 주어
- "~에 있어서", "~에 대하여" 등 불필요한 관용구
- "의"의 연속 사용 (3개 이상 금지)

**인칭 사용**: 3인칭 강하게 선호
- "본 연구는 ~을 분석했다" (O)
- "연구자는 ~을 확인했다" (O)
- "나는 ~을 분석했다" (X)

### English Academic Writing (영어)

**Key principles**:
- Prefer active voice where appropriate
- Keep sentences under 25 words when possible
- Define technical terms on first use
- Use transitions to connect ideas explicitly

**Common issues**:
- Excessive nominalization (turning verbs into nouns)
- Overuse of "there is/are" constructions
- Weak verbs (is, has, does) where stronger verbs exist
- Hedging overload (perhaps, possibly, might, etc.)

## 검토 체크리스트 (Quick Checklist)

글 작성 후 다음을 점검:

- [ ] 모든 문장에 명확한 주어가 있는가
- [ ] 문장 길이가 3단계 기준 내인가 (권장 40자/25w, 최대 60자/30w)
- [ ] 불필요한 형용사/부사를 제거했는가
- [ ] 피동형을 능동형으로 전환했는가
- [ ] 모호한 표현을 구체화했는가
- [ ] "다양한", "여러", "많은" 등을 수치로 대체했는가
- [ ] 접속사 "그리고", "또한"을 최소화했는가
- [ ] 각 단락이 하나의 논점을 다루는가
- [ ] 전문용어를 첫 사용 시 정의했는가
- [ ] 주장이 근거/인용으로 뒷받침되는가
- [ ] 분야별 스타일 가이드를 따르는가
